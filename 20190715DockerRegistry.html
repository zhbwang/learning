安装过程略。

pull地址：192.168.129.115:8084

2. Docker 配置仓库地址

假设 Docker 服务器地址为 192.168.129.137、192.168.129.114。

  • 第一步，在 192.168.129.137 、192.168.129.114 中配置仓库地址。

      # 修改配置文件
  [root@bd129137 ~]# cat /etc/docker/daemon.json
  {
  "insecure-registries":[
      "192.168.129.115:8083","192.168.129.115:8084"
    ]
  }

  # 重启服务，此时原来启动的docker服务器将停止，如果docker镜像启动命令带重启参数会自动重启。
  $ sudo systemctl daemon-reload 
  $ sudo systemctl restart docker

  • 第二步，登录仓库，在本机生成认证密钥。

      [root@bd129137 ~]# docker login 192.168.129.115:8083
  Username: tydic
  Password: tydic
  WARNING! Your password will be stored unencrypted in /root/.docker/config.json.
  Configure a credential helper to remove this warning. See
  https://docs.docker.com/engine/reference/commandline/login/#credentials-store

  Login Succeeded
  [root@bd129137 ~]# docker login 192.168.129.115:8084
  Username: tydic
  Password: tydic

3. Docker 远程仓库使用示例

3.1. 一个简单的Dockerfile

先创建临时目录 docker-demo，并创建 html 文件。

$ mkdir docker-demo
$ cd docker-demo
$ mkdir html
$ echo '<h1>Hello Docker!</h1>' > html/index.html

接下来在当前目录创建一个叫 Dockerfile 的新文件，包含下面的内容：

# cat <<EOF > Dockerfile
FROM nginx
COPY html/* /usr/share/nginx/html
EOF

3.2. 构建 Docker 镜像到仓库

在 docker-demo 目录中执行以下命令：

[[root@bd129137 docker-demo]# docker build -t 192.168.129.115:8084/docker-demo:0.1 .

[root@bd129137 docker-demo]# docker push 192.168.129.115:8084/docker-demo:0.1

3.3. 在另外一台docker服务器pull镜像并执行

在 192.168.129.114 服务器拉取镜像。

[root@bd129114 ~]# docker pull 192.168.129.115:8084/docker-demo:0.1
0.1: Pulling from docker-demo
Digest: sha256:ed7d353aa1007687673bf8a1e6788455bf87f53008bff599d2a2a7bb378a767a

[root@bd129114 ~]# docker images | grep docker-demo

执行服务。

[root@bd129114 ~]# docker run --name docker-demo -d -p 8080:80 192.168.129.115:8084/docker-demo:0.1
148c22d858fe2e0a84098e71ee49ae8f270efb86f016f2cb7bd6bf54ca52ef92
